✨ KEY FEATURES
---------------
1. Context Menu Integration
   → Right-click selected text
   → "Add to Route Optimizer" appears
   → Address saved to pending stops

2. Manual Address Entry
   → Text input field in popup
   → Press Enter or click Add button
   → Addresses stored locally

3. Automatic Page Scanning
   → Click "Scan Page for Addresses"
   → Detects US address patterns
   → Adds multiple addresses at once

4. Address Highlighting
   → Visual indicators on detected addresses
   → Hover effects with location pin emoji
   → Click highlighted address to add

5. Batch Management
   → View all pending stops
   → Remove individual stops (× button)
   → Clear all stops at once
   → Numbered list with timestamps

6. Badge Counter
   → Shows pending stop count on icon
   → Red badge with white text
   → Updates in real-time

7. Configurable App URL
   → Settings panel (collapsible)
   → Default: http://localhost:5173
   → Change for production deployment
   → Saves preferences locally

8. Smart URL Integration
   → Opens Route Optimizer in new tab
   → Passes stops via URL query parameter
   → Format: ?stops=ADDRESS1|ADDRESS2|ADDRESS3
   → Pipe-separated, URI-encoded

🔧 TECHNICAL DETAILS
--------------------
Manifest Version:     V3 (modern Chrome extension standard)
Permissions:          contextMenus, activeTab, storage, scripting
Host Permissions:     <all_urls> (for page scanning)
Storage:              chrome.storage.local (no external servers)
Content Security:     No inline scripts, CSP compliant

Address Pattern:
  • Matches: [Number] [Street] [Type], [City], [State] [ZIP]
  • Example: "123 Main St, Springfield, IL 62701"
  • Supports: Street, Ave, Rd, Blvd, Lane, Dr, etc.

URL Format:
  • Query param: stops
  • Separator: pipe (|)
  • Encoding: URI component encoding
  • Example: APP_URL?stops=123%20Main%20St|456%20Oak%20Ave

Storage Structure:
  {
    pendingStops: [
      { address: "...", addedAt: 1710824400000 }
    ],
    appUrl: "http://localhost:5173"
  }

📦 INSTALLATION
---------------
1. Open Chrome → chrome://extensions/
2. Enable "Developer mode" (top-right toggle)
3. Click "Load unpacked"
4. Select the chrome-extension folder
5. Pin the extension icon to toolbar

🚀 USAGE WORKFLOW
-----------------
1. Browse to a page with addresses
2. Click extension icon
3. Options:
   a) Scan page automatically
   b) Highlight and click addresses
   c) Enter addresses manually
   d) Right-click and add selected text
4. Review pending stops in popup
5. Click "Send to Route Optimizer"
6. Route Optimizer opens with pre-loaded stops

🔗 INTEGRATION POINTS
---------------------
The Route Optimizer web app needs to:

1. Parse URL query parameter on load:
   const params = new URLSearchParams(window.location.search);
   const stops = params.get('stops')?.split('|') || [];

2. Handle decoded addresses:
   stops.forEach(address => {
     // Add to route optimizer UI
   });

3. (Optional) Send confirmation back:
   window.postMessage({ type: 'stopsReceived', count: stops.length }, '*');
